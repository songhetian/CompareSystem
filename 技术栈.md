# Electron\+React 离线桌面应用开发文档

# 1\. 项目概述

## 1\.1 项目定位

本项目基于 Electron \+ React 开发**纯本地离线桌面应用**，全程无需搭建后端服务器、无需网络支持、不启动 HTTP 服务。所有业务数据、配置数据均本地化存储，依托 Electron 主进程能力操作本地 SQLite 数据库，通过 IPC 实现前后进程通信，满足桌面端离线独立运行需求。

## 1\.2 核心特性

- 完全离线运行，无网络、无服务端依赖

- 本地 SQLite 数据库持久化存储业务数据

- 进程隔离安全架构，符合 Electron 官方安全规范

- 轻量化、工程化、类型安全的前端开发体系

- 适配桌面端交互的 UI 设计与状态管理方案

# 2\. 最终技术栈（定稿）

## 2\.1 核心基础框架

- **Electron**：桌面应用容器，提供主进程、渲染进程双进程架构，兼容跨平台打包运行

- **Electron\-Vite**：项目构建工具，替代传统 Webpack，实现极速热更新、快速打包

- **React 18**：前端 UI 渲染核心框架

- **TypeScript**：全项目类型约束，保障代码安全性、可维护性

## 2\.2 路由体系

- **react\-router\-dom v6**：实现多页面路由跳转、导航管理、路由守卫，支撑桌面端多页面业务场景

## 2\.3 状态管理体系

- **Zustand**：轻量全局状态管理，统一管理 UI 全局状态（主题、窗口配置、选中状态、全局参数等），替代笨重的 Redux，无冗余模板代码

## 2\.4 异步数据管理

- **TanStack Query（React Query）**：统一管理 IPC 异步请求、本地数据库查询，自动处理 loading、error、缓存、重试、数据刷新，简化异步业务代码

## 2\.5 UI 与样式体系

- **Arco Design（字节跳动 UI 库）**：桌面端适配优质组件库，提供完整的表单、弹窗、表格、导航等业务组件

- **TailwindCSS 3**：原子化 CSS 方案，快速构建自定义样式，无需编写繁琐自定义 CSS

- **clsx**：辅助拼接动态 class 类名，适配 Tailwind 动态样式场景

## 2\.6 本地数据存储体系

## 2\.7 进程通信体系

- **lodash**：通用数据处理工具库，处理对象、数组、防抖节流等场景

# 3\. 关键技术选型说明（核心答疑）

## 3\.1 Fastify 选型：禁止使用

Fastify 为 HTTP 服务框架，用于搭建后端接口服务。本项目为纯离线桌面应用，无需网络请求、无需接口服务，Electron 主进程本身具备完整 Node\.js 能力，通过 IPC 即可完成所有本地业务交互。引入 Fastify 会造成端口占用、包体积增大、性能冗余，**项目全程禁止使用**。

## 3\.2 Prisma 选型：可选，推荐中大型项目使用

Prisma 是适配 SQLite 的优质 ORM 框架，可实现数据库结构可视化管理、自动迁移、全类型校验，彻底规避原生 SQL 手写错误、无类型约束的问题。

- 小型简单项目：无需引入，直接使用 better\-sqlite3 原生 SQL 即可，轻量化高效

- 中大型迭代项目：推荐引入，提升数据库层可维护性与迭代效率

## 3\.3 核心分工原则（标准化规范）

- **React Router**：负责页面路由跳转、导航结构管理

- **Zustand**：负责全局 UI 状态、应用配置状态管理

- **React Query**：负责 IPC 异步请求、数据库数据缓存与状态管理

- **Electron IPC**：负责渲染进程与主进程的安全通信

- **better\-sqlite3/Prisma**：负责本地数据库数据持久化读写

# 5\. 核心核心功能实现规范

## 5\.1 Electron 进程通信规范（IPC）

### 5\.1\.1 核心原则

- 严格遵循 Electron 安全规范：开启 **contextIsolation**、关闭 **nodeIntegration**

- 所有本地高危操作（数据库读写、文件读写、系统操作）统一在主进程执行

- 渲染进程仅通过 preload 暴露的 API 调用主进程能力，禁止直接操作 Node 原生 API

### 5\.1\.2 通信流程

React 渲染进程（发起请求）→ preload 脚本（中转调用）→ 主进程（处理业务、操作数据库）→ 原路返回结果

## 5\.2 数据分层规范

- **持久层**：主进程 better\-sqlite3/Prisma 操作 SQLite 数据库，存储核心业务数据

- **配置层**：electron\-store 存储应用轻量配置数据

- **异步层**：React Query 封装 IPC 请求，管理数据加载、缓存、刷新

- **状态层**：Zustand 管理全局 UI 状态，与服务端数据解耦

# 6\. 开发注意事项

## 6\.1 安全规范

- 禁止开启 nodeIntegration，禁止在渲染进程使用 Node 原生 API

- 所有 IPC 通信统一封装，禁止随意暴露高危系统能力

- 数据库文件存储在系统用户目录（app\.getPath\('userData'\)），避免打包后路径丢失

## 6\.2 性能规范

- 数据库读写统一在主进程批量处理，避免频繁 IPC 通信造成卡顿

- 通过 React Query 缓存本地数据，减少重复数据库查询

- Zustand 状态按需更新，避免全局无效重渲染

## 6\.3 离线适配规范

- 所有业务逻辑、数据读写不依赖网络请求，全程本地化执行

- 应用启动无需校验网络，打包后可直接离线安装运行

- 数据自动持久化，关闭应用、重启电脑数据不丢失
