## 🚀 快速开始

### 步骤 1: 准备历史数据

**前置条件**: 至少有一个项目，包含 30 天以上的历史数据

```
1. 打开"业务数据导入"页面
2. 创建项目（如：天猫旗舰店）
3. 导入历史业务数据
   - 日期、销售额、人数、咨询量、转化率
```

**推荐**: 数据越多，预测越准确！

---

### 步骤 2: 使用智能预测

#### 2.1 打开测算工作室

```
主页 → 智能人力测算 → 步骤 1: 目标设定
```

#### 2.2 输入目标

```
1. 选择测算模板（可选）
2. 选择驱动方式: 按销售额算
3. 输入目标销售额: 例如 2000 万
4. 选择活动力度（可选）
```

#### 2.3 参考历史数据

```
点击"参考历史数据"按钮
  ↓
选择历史项目: 天猫旗舰店
  ↓
查看统计信息:
  • 平均销售额: 1234.5 万
  • 平均人数: 42 人
  • 人均效率: 29.39 万/人
  ↓
预测结果自动显示:
  约需 69 人
```

#### 2.4 查看详细预测

```
点击"智能预测人力需求"按钮
  ↓
显示详细分析弹窗:
  • 基于 90 条历史数据
  • 各项统计指标
  • 预测计算公式
  • 预测结果: 69 人
```

#### 2.5 继续测算

```
关闭历史数据弹窗
  ↓
点击"下一步"
  ↓
完成后续步骤
  ↓
获得完整测算结果
```

## 💡 使用技巧

### 技巧 1: 选择合适的参考项目

```
✅ 推荐:
  - 业务模式相似的项目
  - 数据量大的项目（30天+）
  - 近期数据（最近 90 天）

❌ 避免:
  - 业务模式差异大的项目
  - 数据量少的项目（<10天）
  - 过时的历史数据
```

### 技巧 2: 理解预测结果

```
预测人数 = 目标销售额 ÷ 历史人均效率

示例:
  目标: 2000 万
  历史人均效率: 29.39 万/人
  预测: 2000 ÷ 29.39 = 68.05 ≈ 69 人
```

### 技巧 3: 合理使用预测

```
预测结果是参考，不是绝对值！

建议:
  1. 预测结果 ± 10-15% 作为实际配置
  2. 考虑活动力度的影响
  3. 结合业务经验调整
  4. 预留应急人员
```

### 技巧 4: 快速应用历史数据

```
在历史数据表格中:
  1. 找到目标日期
  2. 点击"应用"按钮
  3. 销售额自动填入
  4. 继续测算流程
```

## ⚠️ 注意事项

### 1. 数据质量很重要

```
✅ 确保历史数据:
  - 真实准确
  - 完整记录
  - 及时更新
  - 持续积累

❌ 避免:
  - 虚假数据
  - 数据缺失
  - 长期不更新
  - 异常数据未标记
```

### 2. 预测有适用场景

```
✅ 适合:
  - 业务稳定
  - 模式相似
  - 数据充足
  - 常规活动

❌ 不适合:
  - 业务突变
  - 首次尝试
  - 数据极少
  - 特殊情况
```

### 3. 结合业务判断

```
智能预测 + 业务经验 = 最佳决策

不要:
  ❌ 完全依赖预测
  ❌ 忽视实际情况
  ❌ 机械照搬

要做:
  ✅ 预测作为参考
  ✅ 考虑业务变化
  ✅ 适当调整
  ✅ 预留弹性
```
