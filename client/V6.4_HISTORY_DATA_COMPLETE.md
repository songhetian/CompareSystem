## 🏗️ 技术架构

### 数据库设计

#### history_projects 表
```sql
CREATE TABLE history_projects (
  id INTEGER PRIMARY KEY,
  project_name TEXT NOT NULL,      -- 项目名称
  description TEXT,                -- 项目描述
  is_active INTEGER DEFAULT 1,     -- 是否启用（软删除）
  create_time TIMESTAMP,
  update_time TIMESTAMP
);
```

#### history_biz_data 表
```sql
CREATE TABLE history_biz_data (
  id INTEGER PRIMARY KEY,
  project_id INTEGER NOT NULL,     -- 关联项目
  data_date TEXT NOT NULL,
  sales_volume REAL,               -- 销售额
  actual_staff INTEGER,            -- 人数
  actual_consult REAL,             -- 咨询量
  conversion_rate REAL,            -- 转化率
  remark TEXT,
  create_time TIMESTAMP,
  FOREIGN KEY (project_id) REFERENCES history_projects(id)
    ON DELETE CASCADE              -- 级联删除
);
```

**设计特点：**
- ✅ 外键约束确保数据完整性
- ✅ 级联删除：删除项目自动删除数据
- ✅ 软删除：项目标记 `is_active=0`
- ✅ 索引优化：project_id 和 data_date

```typescript
// 获取数据（可按项目过滤）
window.api.getHistoryData(projectId?: number, limit?: number): Promise<HistoryDataRecord[]>

  projectId: 1,
  date: '2026-06-01',

window.api.batchHistoryData(projectId, records): Promise<{success: boolean, count: number}>

// 批量删除
window.api.deleteHistoryData([1, 2, 3]): Promise<any>
```

## 📊 数据流转

### 导入流程
```
用户上传 Excel
  ↓
FileReader 读取文件
  ↓
XLSX.read 解析
  ↓
sheet_to_json 转换
  ↓
数据校验和转换
  ↓
window.api.batchHistoryData(projectId, records)
  ↓
IPC → 主进程
  ↓
SQLite 事务批量插入
  ↓
返回成功/失败
  ↓
刷新表格显示
```

### 导出流程
```
点击导出按钮
  ↓
获取当前项目数据
  ↓
格式化数据
  ↓
XLSX.utils.json_to_sheet
  ↓
创建工作簿
  ↓
XLSX.writeFile
  ↓
下载到本地
```
