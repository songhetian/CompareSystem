## 🎯 功能概述

同比环比分析功能允许用户分析历史数据的时间维度变化，通过对比去年同期（同比）或上一周期（环比）的数据，识别业务增长趋势、季节性规律和周期性波动。

### 核心价值

1. **同比分析（Year-over-Year）** - 消除季节性影响，反映真实年度增长
2. **环比分析（Month-over-Month）** - 快速响应短期变化
3. **趋势预测** - 基于历史增长率预测未来走势
4. **数据驱动决策** - 量化评估业务发展情况

## ✨ 功能特性

### 1. 分析类型选择

**同比分析（YoY）**
- 定义：与去年同期对比
- 示例：2026年1月 vs 2025年1月
- 适用场景：
  - 评估年度目标达成情况
  - 分析长期业务发展趋势
  - 消除季节性因素影响

**环比分析（MoM）**
- 定义：与上一个周期对比
- 示例：2月 vs 1月，第10周 vs 第9周
- 适用场景：
  - 快速响应市场变化
  - 监控短期业务波动
  - 及时调整运营策略

### 2. 统计周期选择

**按月统计**
- 适合中长期趋势分析
- 数据平滑，波动较小
- 适合制定季度/年度计划

**按周统计**
- 适合短期快速响应
- 更敏感地捕捉变化
- 适合周度运营调整

### 3. 增长率指标

**销售额增长率**
```typescript
growth = ((current - previous) / previous) * 100
```

**人数增长率**
```typescript
staffGrowth = ((currentStaff - previousStaff) / previousStaff) * 100
```

**效率增长率** ⭐ 最关键指标
```typescript
efficiencyGrowth = ((currentEfficiency - previousEfficiency) / previousEfficiency) * 100
```

### 4. 可视化展示

**增长率趋势图**
- 3条曲线：销售额、人数、效率
- 不同颜色区分指标
- 显示正负增长方向（↑↓）

**数据明细表格**
- 周期列（2026-01, 2026-W10等）
- 增长率列（带颜色和方向图标）
- 当前值列（销售额、人数、效率）
- 对比基准列（去年同期/上一周期）

### 2. 核心算法

**同比计算 (calculateYoY)**

```typescript
// 1. 按周/月分组历史数据
const periodMap = groupDataByPeriod(data, analysisPeriod);

// 2. 计算每个周期的平均值
const periods = calculatePeriodAverages(periodMap);

// 3. 查找去年同期数据
const comparisons = periods.map(current => {
  const lastYear = findSamePeriodLastYear(current, periods);
  return {
    period: current.period,
    current,
    lastYear,
    salesGrowth: (current.avgSales - lastYear.avgSales) / lastYear.avgSales * 100,
    staffGrowth: (current.avgStaff - lastYear.avgStaff) / lastYear.avgStaff * 100,
    efficiencyGrowth: (current.avgEfficiency - lastYear.avgEfficiency) / lastYear.avgEfficiency * 100
  };
});

return comparisons;
```

**环比计算 (calculateMoM)**

```typescript
// 1. 按周/月分组并排序
const periods = groupAndSortData(data, analysisPeriod);

// 2. 计算与上一周期的对比
const comparisons = periods.slice(1).map((current, index) => {
  const previous = periods[index];
  return {
    period: current.period,
    current,
    previous,
    salesGrowth: (current.avgSales - previous.avgSales) / previous.avgSales * 100,
    staffGrowth: (current.avgStaff - previous.avgStaff) / previous.avgStaff * 100,
    efficiencyGrowth: (current.avgEfficiency - previous.avgEfficiency) / previous.avgEfficiency * 100
  };
});

return comparisons;
```

### 3. 数据处理

**周期分组**
```typescript
// 按月分组
const monthKey = dayjs(date).format('YYYY-MM');  // 2026-01

// 按周分组
const weekKey = dayjs(date).format('YYYY-[W]WW');  // 2026-W10
```

**平均值计算**
```typescript
const avgSales = records.reduce((sum, r) => sum + r.sales_volume, 0) / records.length;
const avgStaff = records.reduce((sum, r) => sum + r.actual_staff, 0) / records.length;
const avgEfficiency = avgSales / avgStaff;
```

## 📊 数据需求

### 最低数据要求

**同比分析**
- 至少需要跨2年的数据
- 每年至少有1个相同月份/周的数据
- 示例：2025-01 和 2026-01

**环比分析**
- 至少需要2个连续周期的数据
- 按月：至少2个月
- 按周：至少2周

### 数据质量要求

- 数据完整性：每个周期需要至少1条记录
- 有效数据：actual_staff > 0 才计算效率
- 时间连续性：周期之间不应有过大间隔

## 🎓 最佳实践

### 分析建议

**同比分析适用于**:
- 制定年度目标和预算
- 评估长期战略效果
- 向管理层汇报业绩

**环比分析适用于**:
- 日常运营监控
- 快速响应市场变化
- 周/月度复盘会议

### 解读技巧

**正增长**:
- 销售额↑ 人数↑ 效率→ : 规模扩张型增长
- 销售额↑ 人数→ 效率↑ : 效率驱动型增长 ⭐最佳
- 销售额↑ 人数↑ 效率↓ : 粗放型增长，需优化

**负增长**:
- 销售额↓ 人数↓ 效率→ : 收缩调整期
- 销售额↓ 人数→ 效率↓ : 业务下滑，需警惕
- 销售额↑ 人数↑↑ 效率↓ : 人力浪费，需优化
