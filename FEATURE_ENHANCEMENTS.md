### 3. 大促方案管理系统 ✅

#### 3.1 数据库表结构
```sql
-- 大促方案表
CREATE TABLE promotion_schemes (
    id INTEGER PRIMARY KEY,
    scheme_name TEXT NOT NULL,
    traffic_factor REAL NOT NULL,
    description TEXT,
    create_time TIMESTAMP,
    update_time TIMESTAMP
);

-- 默认方案
- S级大促(双11/618): 2.8倍
- A级活动(会员日): 1.9倍

```

#### 3.2 功能特性

### 4. 时间段配置系统 ✅

#### 4.1 数据库表结构
```sql
-- 时间段配置表
CREATE TABLE time_slot_configs (
    id INTEGER PRIMARY KEY,
    phase TEXT NOT NULL,  -- presale/midsale/aftersale
    start_hour INTEGER NOT NULL,
    end_hour INTEGER NOT NULL,
    burst_factor REAL NOT NULL,
    staff_ratio REAL NOT NULL,
    create_time TIMESTAMP
);
```

#### 4.2 功能特性
- ✅ 为售前/售中/售后分别配置多个时间段
- ✅ 每个时间段可设置：
  - 开始时间（0-23点）
  - 结束时间（1-24点）
  - 爆发倍数（流量放大系数）
  - 人力配比（该时段需要的人力比例）
- ✅ 支持添加、编辑、删除时间段
- ✅ 根据时间段动态计算不同时刻的客服需求

## 📝 使用说明

### 1. 管理大促方案
1. 进入"人力预算智能工作台"
2. 点击"⚙️ 管理大促方案"按钮
3. 添加自定义方案（名称、系数、说明）
4. 删除不需要的方案

### 2. 保存历史方案
1. 完成一次人力测算
2. 点击"💾 保存为历史方案"
3. 输入方案名称
4. 方案自动保存到数据库

### 3. 加载历史方案
1. 点击"📂 加载历史方案"
2. 从列表中选择要加载的方案
3. 点击"📥 加载"
4. 所有参数和结果自动恢复

## ⚠️ 注意事项

1. 数据库迁移必须在使用新功能前执行
2. 旧的大促参数已被清理，请使用新的方案管理系统
3. 历史方案以JSON格式保存，确保参数兼容性
