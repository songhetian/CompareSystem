# 快速启动指南

## 🚀 首次使用

### 1. 运行数据库迁移
```bash
python database/migrations.py
```

这将创建新的数据库表并插入默认数据。

### 2. 启动应用
```bash
python main.py
```

## 📖 功能使用指南

### 一、人力预算测算

#### 基础测算流程
1. 进入"📊 人力预算测算"页面
2. 输入销售总额（万元）
3. 选择日期范围
4. 选择测算场景（大促方案）
5. 勾选参与的班次
6. 点击"⚡ 执行全量测算"

#### 查看结果
- **建议总人力**：系统推荐的总人数
- **预估日均咨询**：每天的咨询量
- **日均需求工时**：每天需要的工作小时数
- **关键洞察**：详细的岗位配置和业务量分布

### 二、大促方案管理

#### 添加自定义方案
1. 在预算测算页面点击"⚙️ 管理大促方案"
2. 输入方案名称（如"C级活动(品牌日)"）
3. 输入流量系数（如1.5，表示流量是平时的1.5倍）
4. 输入方案说明
5. 点击"➕ 添加方案"

#### 使用方案
- 在"测算场景"下拉框中选择你创建的方案
- 系统会自动应用对应的流量系数

#### 删除方案
- 在方案列表中点击"🗑️"按钮

### 三、时间段配置

#### 配置爆发时段
1. 点击侧边栏"⏰ 时间段配置"
2. 选择要配置的阶段（售前/售中/售后）
3. 输入时间段信息：
   - **开始时间**：如10（表示10点）
   - **结束时间**：如12（表示12点）
   - **爆发倍数**：如1.9（表示该时段流量是平时的1.9倍）
   - **人力配比**：如1.0（表示该时段需要100%的人力）
4. 点击"➕"添加

#### 配置示例

**售前咨询高峰**
- 上午咨询高峰：10:00-12:00，爆发倍数1.9
- 下午咨询高峰：14:00-16:00，爆发倍数1.5

**售中服务高峰**
- 午间改单高峰：12:00-14:00，爆发倍数2.0
- 下午催单高峰：15:00-17:00，爆发倍数2.3

**售后服务高峰**
- 晚间退货高峰：20:00-22:00，爆发倍数2.6
- 深夜投诉高峰：22:00-24:00，爆发倍数1.8

#### 删除时间段
- 点击时间段右侧的"🗑️"按钮

### 四、历史方案管理

#### 保存当前方案
1. 完成一次人力测算
2. 点击"💾 保存为历史方案"
3. 输入方案名称（如"618大促预算方案"）
4. 点击确定

#### 加载历史方案
1. 点击"📂 加载历史方案"
2. 从列表中选择要加载的方案
3. 点击"📥 加载"
4. 所有参数和结果自动恢复

#### 方案用途
- 对比不同时期的人力需求
- 快速恢复历史配置
- 作为新方案的参考基准

### 五、24小时人力曲线

#### 查看曲线
1. 完成人力测算
2. 点击"📈 查看24小时人力曲线"

#### 曲线解读
- **堆叠柱状图**：显示每小时的售前/售中/售后人力需求
- **峰值时段**：人力需求最高的时间段
- **峰值人力**：峰值时段需要的总人数
- **平均人力**：24小时的平均人力需求

#### 应用场景
- 优化排班计划
- 识别人力缺口
- 调整班次时间
- 合理分配人力资源

### 六、参数配置

#### 调整参数
1. 点击侧边栏"⚙️ 全局参数配置"
2. 找到要调整的参数
3. 拖动滑块或直接输入数值
4. 点击"🚀 同步并应用新基准"

#### 重要参数说明

**基础业务参数**
- **平均客单价**：影响订单量计算
- **日访客数**：基础流量
- **全日高峰系数**：全天流量波动
- **安全冗余系数**：预留备用人力

**漏斗转化参数**
- **访客转售前咨询率**：有多少访客会咨询
- **咨询转下单率**：咨询后有多少会下单
- **下单转付款率**：下单后有多少会付款
- **付款转售后率**：付款后有多少会产生售后

**岗位效能参数**
- **单次处理时长**：处理一次业务需要的时间
- **工作饱和度**：有效工作时间占比

## 💡 使用技巧

### 1. 大促前准备
1. 配置大促方案（设置流量系数）
2. 配置时间段（根据历史数据设置爆发时段）
3. 调整参数（根据实际情况微调）
4. 执行测算并保存方案

### 2. 日常运营
1. 使用"日常运营"方案（流量系数1.0）
2. 根据实际业务量调整参数
3. 定期保存方案作为基准

### 3. 方案对比
1. 保存多个不同场景的方案
2. 加载不同方案查看人力需求差异
3. 选择最优方案

### 4. 排班优化
1. 查看24小时人力曲线
2. 识别峰值时段
3. 调整班次时间匹配峰值
4. 确保峰值时段人力充足

## ⚠️ 常见问题

### Q1: 计算结果不合理？
**A:** 检查以下参数：
- 客单价是否正确
- 漏斗转化率是否符合实际
- 时间段配置是否合理

### Q2: 如何恢复默认参数？
**A:** 删除数据库文件后重新运行迁移脚本

### Q3: 时间段配置不生效？
**A:** 确保：
- 时间段已保存
- 重新执行了测算
- 时间段没有重叠冲突

### Q4: 历史方案加载失败？
**A:** 可能是参数结构变化，建议重新保存方案
